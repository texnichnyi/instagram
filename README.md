## Резюме Архітектурних Рішень Page Object Model (POM)


### 1. Ієрархія та Успадкування (Scalability)

Створено чітку трирівневу ієрархію базових класів, яка забезпечує чистоту коду:

* **`BasePage`:** Містить лише універсальну інфраструктуру, доступну всім користувачам (залогованим та незалогованим): `WebDriver`, обгортки для взаємодії (`find/click/fill`).
* **`BaseAuthPage`:** Успадковує від `BasePage`. Додає функціональність, специфічну для **залогованого користувача** (наприклад, елементи нижнього меню). Це ізолює логіку стану користувача.
* **Конкретні Page Objects:** (`HomePage`, `LoginPage`, `SignUpPage`): Успадковують від відповідного базового класу.

### 2. Стратегія Надійного Очікування (Wait Strategy)

Для стабільної роботи з динамічними сторінками використовується `wait_for_loading`:

* **Явне Очікування :** Метод `self.wait_for_loading(element_locator=KEY_ELEMENT)` використовується у конструкторах конкретних сторінок. Це гарантує, що критично важливий елемент з'явився після рендерингу.

### 3. Управління Потоком

Для створення лінійних та читабельних сценаріїв використовуються:

* Методи дії, що призводять до навігації (наприклад, `login()`), **повертають об'єкт наступної сторінки**.
* **Приклад:** `login_page.login(user, pass)` повертає об'єкт `HomePage`.
* **Обґрунтування:** Робить тестові сценарії лінійними та читабельними (`home_page = login_page.login(..)`), а також перекладає відповідальність за **валідацію успішного переходу** на конструктор нового об'єкта (`HomePage.__init__`), де викликається `wait_for_loading`.

### 4. Композиція та Компоненти

Використовується чітке розділення на Page Objects та Component Objects:

* **Композиція:** `LoginPage` вбудовує `LoginForm` (Композиція).
* **Колекція Компонентів:** `HomePage` використовує `get_all_posts()` для створення та управління колекцією `PostComponent`.
* **Ледача Ініціалізація (Lazy Init):** Компоненти, доступні на багатьох сторінках (`Footer`, `BottomPanel`), створюються лише при першому зверненні через `@property` (наприклад, `@property footer`). Це підвищує продуктивність, оскільки об'єкти не створюються, якщо вони не використовуються в тесті.
